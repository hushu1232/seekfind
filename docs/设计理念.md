# 求问 — 设计理念

> 核心设计原则与技术选型依据。

---

## 一、核心面向：Microsoft Edge

### 为什么选 Edge

| 维度 | Edge 优势 |
|------|----------|
| **用户基数** | Windows 11 默认浏览器，企业用户占比高 |
| **Chromium 内核** | 与 Chrome 共享扩展生态，兼容性 100% |
| **侧边栏 API** | 原生 sidePanel 支持，体验优于 Chrome |
| **企业集成** | Azure AD / Intune 集成，适合企业部署 |
| **性能** | 内存管理优于 Chrome，长时间运行更稳定 |
| **隐私** | 内置隐私保护，与求问"隐私优先"理念一致 |

### 浏览器优先级

```
1. Edge      ← 核心面向，优先测试
2. Chrome    ← 完全兼容（同 Chromium 内核）
3. Firefox   ← 社区支持（MV2 兼容层）
```

### 构建命令

```bash
npm run build          # 默认构建 Edge 版本
npm run build:edge     # 明确指定 Edge
npm run build:chrome   # Chrome 版本
npm run build:firefox  # Firefox 版本
```

### 加载方式

```
Edge:  edge://extensions → 开发者模式 → 加载已解压 → extension/dist
Chrome: chrome://extensions → 开发者模式 → 加载已解压 → extension/dist
Firefox: about:debugging → 此 Firefox → 加载临时附加组件
```

## 二、隐私优先

### 设计原则

1. **数据不出本机** — 所有 AI 推理（Ollama）、向量检索（Chroma）、记忆存储（SQLite）均在本地运行
2. **采集即脱敏** — 邮箱/手机/密码/密钥在 Content Script 层脱敏后再传输
3. **用户可控** — 监控/录制/历史学习均可独立开关
4. **无追踪** — 不使用 Google Analytics、不发送遥测数据

### 数据流

```
用户操作 → Content Script（脱敏）→ Service Worker → WS → 后端（本地）
                                                              ↓
                                                         Chroma / SQLite（本地）
                                                              ↓
                                                         Ollama（本地推理）
```

## 三、AI 原生

### 设计原则

1. **AI 实时推理** — 不需要预录制，LLM 实时理解页面并生成指引
2. **自学习飞轮** — 元素指纹越用越准，成功案例自动复用
3. **渐进增强** — 本地 7B → 本地 14B → 云端 API，按需升级
4. **混合检索** — 向量 + BM25 + RRF，兼顾语义和关键词

### Agent 架构

```
用户提问 → Supervisor（任务拆解）→ 并行 Workers → 聚合 → LLM 回复
              ↓ (降级)
         原单 Agent
```

## 四、渐进式披露

### 设计原则

1. **零配置可用** — 内置 220 条知识库，首次安装即可使用
2. **按需导入** — 用户可导入自己的文档扩充知识
3. **新用户引导** — 悬浮球前 3 次自动显示提示
4. **快捷菜单** — 右键/长按唤出常用操作

---

## 五、性能优先

### 设计原则

1. **懒加载** — 工具按需加载，减少启动开销
2. **并行执行** — 多 Worker 并行，复合指令耗时 < 串行 * 0.65
3. **上下文管理** — Token 计数 + 自动截断，防止超限
4. **动画优化** — CSS 动画 + requestAnimationFrame，60fps 流畅

---

## 六、可扩展性

### 设计原则

1. **MCP Server** — 暴露 7 个工具，供其他 AI Agent 调用
2. **工具注册表** — 新增工具只需添加到 `_TOOL_REGISTRY`
3. **模块化架构** — 前后端模块独立，可单独测试
4. **双浏览器兼容** — Chrome/Edge 共用代码，Firefox 独立 manifest
