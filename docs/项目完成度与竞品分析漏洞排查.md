## 二、同类项目对比分析

### 2.1 竞品矩阵

| 项目 | 类型 | Stars | 核心能力 | 求问对比 |
|------|------|-------|---------|---------|
| **WalkMe** | DAP 商业 | - | 预录制操作流 + 分析 | 求问: AI 实时推理，无需预录制 |
| **Whatfix** | DAP 商业 | - | 交互式引导 + 内容管理 | 求问: 本地化，数据不出本机 |
| **Scribe** | AI 录制 | - | 录制操作自动生成指南 | 求问: 有录制 + AI 推理双模式 |
| **Harpa AI** | 浏览器扩展 | 100K+ | ChatGPT + 自动化 | 求问: 专注操作引导，非通用 AI |
| **Monica AI** | 浏览器扩展 | 200K+ | ChatGPT 副驾驶 | 求问: 本地模型，隐私优先 |
| **agent-browser** | CLI 工具 | 34K+ | AI 浏览器自动化 | 求问: 已集成其核心能力 |
| **Scrapling** | 爬取框架 | 55K+ | 自适应爬取 + 反检测 | 求问: 已集成其 Fetcher |

### 2.2 求问的核心优势

| 优势 | 说明 | 竞品对比 |
|------|------|---------|
| **完全本地化** | Ollama + Chroma，数据不出本机 | WalkMe/Whatfix/Monica 均需云端 |
| **AI 实时推理** | 不需要预录制，LLM 实时理解页面 | WalkMe/Scribe 需要人工录制 |
| **混合检索** | 向量 + BM25 + RRF，精准定位 | 多数竞品只用关键词匹配 |
| **元素指纹系统** | SQLite 存储 + 模糊匹配，越用越准 | 竞品无此自学习能力 |
| **无障碍树快照** | @eN 引用，AI 友好的页面理解 | agent-browser 同级能力 |
| **MCP Server** | 7 个工具，可被其他 Agent 调用 | 竞品无此开放能力 |
| **多浏览器支持** | Chrome/Firefox/Edge 双 manifest | 多数竞品仅 Chrome |
| **语音交互** | ASR + TTS + 唤醒词 | WalkMe/Whatfix 无语音 |

### 2.3 求问的劣势

| 劣势 | 说明 | 竞品对比 |
|------|------|---------|
| **模型能力天花板** | 7B 本地模型 < GPT-4o | Monica/Harpa 用 GPT-4o |
| **知识库冷启动** | 新用户无文档时回答质量低 | WalkMe 有预置行业模板 |
| **视觉定位精度** | moondream2 1.6B 精度有限 | 竞品用 GPT-4V 更准 |
| **无分析仪表盘** | 无用户行为分析 | WalkMe/Whatfix 有完整分析 |
| **部署门槛** | 需 Docker + Ollama + Chrome | 竞品一键安装 |
| **移动端不支持** | 仅桌面浏览器 | 部分竞品支持移动端 |
| **无团队协作** | 单机版，无多用户管理 | WalkMe/Whatfix 有企业管理 |

## 三、项目漏洞排查

### 3.1 安全漏洞

| 编号 | 漏洞 | 严重性 | 位置 | 说明 |
|------|------|--------|------|------|
| **V1** | XSS 风险 — innerHTML 使用 | 🟡 中 | `panel.ts:287,316,374` | 使用 `innerHTML` 渲染用户内容，虽有 `escapeHtml` 但部分位置直接拼接 |
| **V2** | WebSocket 明文传输 | 🟡 中 | `constants.ts:10` | 使用 `ws://` 而非 `wss://`，本地可接受但远程部署需修复 |
| **V3** | 权限过度 | 🟡 中 | `manifest.json` | `<all_urls>` + `scripting` 权限过宽，可访问所有网站 |
| **V4** | console 输出泄露 | 🟢 低 | 14 处 | 生产环境应移除 `console.log` |
| **V5** | 硬编码 API Key | 🟢 低 | `agent.py:164` | `api_key="ollama"` 硬编码，虽是本地但不规范 |

### 3.2 架构漏洞

| 编号 | 漏洞 | 严重性 | 说明 | 修复方案 |
|------|------|--------|------|---------|
| **V6** | 无请求速率限制 | 🟡 中 | WS 消息无频率限制，可被刷 | 添加 rate limiter |
| **V7** | 无输入长度限制 | 🟡 中 | 用户输入无长度校验 | 添加 max_length 校验 |
| **V8** | 无错误边界 | 🟡 中 | Content Script 未捕获全局异常 | 添加 window.onerror |
| **V9** | 内存泄漏风险 | 🟡 中 | `activeHighlights[]` 无限增长 | 添加上限 + 自动清理 |
| **V10** | 无超时控制 | 🟡 中 | Agent 工具调用无超时 | 添加 per-tool timeout |
| **V11** | 状态不一致 | 🟢 低 | Service Worker 重启后内存状态丢失 | 持久化到 chrome.storage |

### 3.3 功能漏洞

| 编号 | 漏洞 | 严重性 | 说明 | 修复方案 |
|------|------|--------|------|---------|
| **V12** | 侧边栏刷新丢失状态 | 🟡 中 | 页面刷新后侧边栏重新加载，对话丢失 | 从 storage 恢复完整状态 |
| **V13** | 高亮跨域失效 | 🟡 中 | 跨域导航后高亮 selector 可能失效 | 重新查询或清除失效高亮 |
| **V14** | 语音识别无模型时不提示 | 🟢 低 | ASR 无模型时静默返回空 | 前端提示"需安装 Vosk" |
| **V15** | 知识库无版本管理 | 🟢 低 | 更新知识库后旧索引不自动更新 | 添加版本号 + 增量重建 |
| **V16** | 操作流无持久化 | 🟡 中 | 录制的操作流仅在内存中，重启丢失 | 录制完成后立即写入 Chroma |
| **V17** | 截图无压缩 | 🟢 低 | base64 截图未压缩，WS 传输大 | 添加 JPEG 压缩 + 质量参数 |
