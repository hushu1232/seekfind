## Sprint 1 — 核心修复（阻塞核心功能）

### T1: Agent 引擎接入 LangGraph + 工具调用循环

- **优先级**: P0
- **问题**: agent.py 声称使用 LangGraph，但实际只用了 ChatOpenAI，没有 StateGraph 编排；工具注册了但未被调用
- **目标**: 用户提问 → 意图分类 → LLM + tool_calls → 执行工具 → ToolMessage 回传 → LLM 最终回复
- **改动文件**: `backend/agent.py`, `backend/tools/__init__.py`
- **状态**: ✅ (2026-05-30)
- **改动摘要**: 重写为 LangGraph StateGraph；工具通过 StructuredTool 包装供 ToolNode 使用

### T2: 高亮工具链路打通

- **优先级**: P0
- **问题**: highlight_element.py 返回 JSON 字符串但没被 Agent 调用；_process_guide_steps 靠 LLM 自由格式输出解析 JSON
- **目标**: LLM 输出步骤描述 → Agent 调用 highlight_element 工具 → 工具返回指令 → 通过 WS 下发
- **改动文件**: `backend/agent.py`, `backend/tools/__init__.py`
- **状态**: ✅ (2026-05-30，T1 已解决——LangGraph ToolNode 自动调用所有注册工具)

## Sprint 4 — 性能优化

### T9: 索引构建优化

- **优先级**: P3
- **问题**: 爬取和索引同步；没有增量索引；固定 500 字符分块可能断句
- **目标**: 异步 pipeline + 增量索引 + 智能分块

- **状态**: ✅ (2026-05-30)
- **改动摘要**: 新增 build_from_url_streaming 流水线模式；提取 _index_chunks 公共方法；智能分块已有（段落→句子→字符）

### T10: 前端 Bundle 优化

- **目标**: Three.js 按需引入 + CSS 外部化

- **状态**: ✅ (2026-05-30)
- **改动摘要**: CSS 提取到 styles.css；Vite 配置增加 manualChunks 分离 Three.js；ball.ts 已有动态 import

### T11: 内存管理优化

- **优先级**: P3
- **问题**: ShortTermMemory 用 list 无上限；Chroma 无连接池；前端 chatHistory 无限增长

- **改动文件**: `backend/memory/short_term.py`, `backend/memory/long_term.py`, `backend/config.py`
- **状态**: ✅ (2026-05-30)
- **改动摘要**: ShortTermMemory 已用 deque(maxlen)；LongTermMemory 增加重试逻辑（指数退避）+ 健康检查 + 连接超时配置化；前端 StateStore 已限制 100 条

## Sprint 5 — Scrapling 强化（2026-05-30）

基于 Scrapling (⭐55K+) 内核分析，提取 6 大设计模式强化项目。

详细实施计划：[Sprint5-Scrapling强化实施计划.md](Sprint5-Scrapling强化实施计划.md)

### T12: 集成 Scrapling Fetcher

- **优先级**: P0
- **问题**: crawler.py 和 fetch_doc_page.py 用 httpx 直接请求，无法处理 JS 渲染页面；trafilatura 对 SPA 返回空；无反爬能力
- **目标**: 用 Scrapling Fetcher 替换 httpx，支持 JS 渲染 + 自动反爬
- **改动文件**: `backend/indexer/scrapling_fetcher.py`(新建), `backend/indexer/crawler.py`, `backend/tools/fetch_doc_page.py`, `backend/requirements.txt`
- **状态**: ✅ (2026-05-30)
- **改动摘要**: 新建 ScraplingFetcher 适配层；crawler.py 和 fetch_doc_page.py 改用 ScraplingFetcher；降级路径保留 httpx

### T13: 元素指纹存储层

- **优先级**: P1
- **问题**: 元素指纹库已规划但未实现；highlight_element 依赖 LLM 推理 selector，准确率有限
- **目标**: 新建 SQLite 元素指纹存储，参考 Scrapling 的 StorageSystemMixin
- **改动文件**: `backend/memory/fingerprint_storage.py`(新建)
- **状态**: ✅ (2026-05-30)
- **改动摘要**: SQLite + WAL + RLock；URL 域名 + 描述模糊匹配（SequenceMatcher）；success/fail 计数；过期清理

### T14: 指纹自动构建与查找

- **优先级**: P1
- **问题**: highlight_element 没有指纹查找/存储闭环
- **目标**: 定位成功后自动存储指纹，下次直接命中缓存
- **改动文件**: `backend/tools/highlight_element.py`, `backend/tools/__init__.py`, `backend/agent.py`
- **状态**: ✅ (2026-05-30)
- **改动摘要**: highlight_element 增加 page_url 参数 + 指纹查找/存储；selector="auto" 时从指纹库查找；agent.py 注入 fingerprint_storage

### T15: CrawledDoc 增强
