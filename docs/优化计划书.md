## Sprint 1 — 核心修复（阻塞核心功能）

### T1: Agent 引擎接入 LangGraph + 工具调用循环

- **优先级**: P0
- **问题**: agent.py 声称使用 LangGraph，但实际只用了 ChatOpenAI，没有 StateGraph 编排；工具注册了但未被调用
- **目标**: 用户提问 → 意图分类 → LLM + tool_calls → 执行工具 → ToolMessage 回传 → LLM 最终回复
- **改动文件**: `backend/agent.py`, `backend/tools/__init__.py`
- **状态**: ✅ (2026-05-30)
- **改动摘要**: 重写为 LangGraph StateGraph；工具通过 StructuredTool 包装供 ToolNode 使用

### T2: 高亮工具链路打通

- **优先级**: P0
- **问题**: highlight_element.py 返回 JSON 字符串但没被 Agent 调用；_process_guide_steps 靠 LLM 自由格式输出解析 JSON
- **目标**: LLM 输出步骤描述 → Agent 调用 highlight_element 工具 → 工具返回指令 → 通过 WS 下发
- **改动文件**: `backend/agent.py`, `backend/tools/__init__.py`
- **状态**: ✅ (2026-05-30，T1 已解决——LangGraph ToolNode 自动调用所有注册工具)

## Sprint 2 — 质量提升（影响体验质量）

### T3: 检索模块优化

- **优先级**: P1
- **问题**: BM25 是在向量检索结果上二次检索，不是全库 BM25；jieba 没有自定义词典
- **目标**: 向量检索 ∥ 全库 BM25 → RRF 融合
- **改动文件**: `backend/tools/search_docs.py`, `backend/memory/long_term.py`
- **状态**: ⬜

### T4: 测试覆盖补充

- **优先级**: P1
- **问题**: 52 个测试只覆盖纯逻辑函数，核心链路（WS/RRF/Agent 流式）没有测试
- **目标**: 补充 test_ws.py, test_search.py, test_agent_stream.py
- **改动文件**: `backend/tests/`
- **状态**: ⬜

### T5: 知识库扩充

- **优先级**: P1
- **问题**: 92 条指引平均每个产品 2.8 条，覆盖不足；selectors 未验证
- **目标**: 高频产品扩充到 10+ 条，添加 last_verified 字段
- **改动文件**: `backend/knowledge/builtin/`
- **状态**: ⬜

## Sprint 3 — 架构优化（提升可维护性）

### T6: 后端 Schema 规范化

- **优先级**: P2
- **问题**: agent.py 中 page_context 类型是 dict[str, Any]；WS 消息没有 schema 验证
- **目标**: 添加 schemas.py，定义 Pydantic model；WS 消息验证
- **改动文件**: `backend/schemas.py`, `backend/app.py`, `backend/agent.py`
- **状态**: ⬜

### T7: 前端模块拆分

- **优先级**: P2
- **问题**: service_worker.ts (326行) 职责过多；highlight.ts (503行) 是最大单文件
- **目标**: 拆分为独立模块，降低单文件复杂度
- **改动文件**: `extension/background/`, `extension/content/`
- **状态**: ⬜

### T8: 类型安全增强

- **优先级**: P2
- **问题**: 前端 panel.ts 中 payload 类型是 any；Chrome Storage 读写分散
- **目标**: 类型守卫 + 统一 store
- **改动文件**: `extension/common/`, `extension/sidebar/panel.ts`
- **状态**: ⬜

## Sprint 4 — 性能优化

### T9: 索引构建优化

- **优先级**: P3
- **问题**: 爬取和索引同步；没有增量索引；固定 500 字符分块可能断句
- **目标**: 异步 pipeline + 增量索引 + 智能分块
- **改动文件**: `backend/indexer/build_index.py`, `backend/indexer/crawler.py`
- **状态**: ⬜

### T10: 前端 Bundle 优化

- **优先级**: P3
- **问题**: Three.js 全量引入 (~600KB)；index.html 内联 164 行 CSS
- **目标**: Three.js 按需引入 + CSS 外部化
- **改动文件**: `extension/sidebar/ball.ts`, `extension/sidebar/index.html`
- **状态**: ⬜

### T11: 内存管理优化

- **优先级**: P3
- **问题**: ShortTermMemory 用 list 无上限；Chroma 无连接池；前端 chatHistory 无限增长
- **目标**: deque + 连接超时 + 内存上限
- **改动文件**: `backend/memory/short_term.py`, `backend/memory/long_term.py`, `extension/background/service_worker.ts`
- **状态**: ⬜
