# 浏览器兼容性方案

> 分析 Chrome / Edge / Firefox 的 API 差异，制定适配方案。

---

## 一、当前状态

### 1.1 兼容性矩阵

| 功能 | Chrome | Edge | Firefox | 问题 |
|------|--------|------|---------|------|
| Manifest V3 | ✅ | ✅ | ✅ (有限) | Firefox 对 MV3 支持不完整 |
| sidePanel | ✅ | ✅ | ❌ | Firefox 用 sidebar_action |
| service_worker | ✅ | ✅ | ❌ | Firefox 用 background.scripts |
| content_scripts | ✅ | ✅ | ✅ | 完全兼容 |
| chrome.storage | ✅ | ✅ | ✅ | 完全兼容 |
| chrome.tabs | ✅ | ✅ | ✅ | 完全兼容 |
| chrome.runtime | ✅ | ✅ | ✅ | 完全兼容 |
| captureVisibleTab | ✅ | ✅ | ✅ | 完全兼容 |

### 1.2 核心差异

| API | Chrome/Edge | Firefox |
|-----|-------------|---------|
| 侧边栏 | `sidePanel` (MV3) | `sidebar_action` (MV2/MV3) |
| 后台脚本 | `service_worker` | `background.scripts` |
| 权限 | `sidePanel` 权限 | `sidebar_action` 权限 |

---

## 二、适配方案

### 方案：双 manifest 构建

为 Chrome/Edge 和 Firefox 分别生成 manifest.json，共享同一套 TS 代码。

```
extension/
├── manifest.chrome.json    # Chrome/Edge 专用
├── manifest.firefox.json   # Firefox 专用
├── scripts/
│   └── build.js            # 构建脚本，根据 target 选择 manifest
└── ...
```
