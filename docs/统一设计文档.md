# 求问 — 统一设计文档

> 基于 gstack office-hours 方法论重构。2026-06-01

---

## 一、项目定位

**一句话：** 求问是一个本地化的 AI 网页引导助手，用户问"在哪里创建项目"，它直接在页面上高亮给你看。

**核心价值：** 让 AI 实时理解网页内容并引导用户操作，数据不出本机。

---

## 二、问题定义

### 用户痛点

| 痛点 | 现状 | 求问方案 |
|------|------|---------|
| 找不到操作入口 | 看文档/搜索/问人 | AI 实时定位 + 高亮 |
| 文档过时 | 文档与实际页面不一致 | 直接分析页面 DOM |
| 学习成本高 | 新产品上手慢 | 分步引导 + 操作流录制 |
| 隐私担忧 | 云端 AI 看到你的操作 | 全部本地化 |

### 目标用户

- **主要**：需要频繁使用 SaaS 工具的知识工作者
- **次要**：企业内部培训（员工学习新系统）
- **潜在**：老年人/无障碍用户（语音引导）

### 4.2 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| Agent 引擎 | `agent_engine.py` | LangGraph 状态机 + 意图路由 |
| 多 Agent | `agent/` | Supervisor + Workers 并行执行 |
| 工具集 | `tools/` | 12 个工具（检索/高亮/视觉/浏览器） |
| 记忆系统 | `memory/` | 短期(deque) + 长期(Chroma) + 持久化(SQLite) |
| 浏览器控制 | `browser/` + `snapshot.ts` | 无障碍树快照 + @eN 引用 |
| MCP Server | `mcp/` | 7 个工具供外部 Agent 调用 |

### 4.3 数据流

```
用户输入 → 意图分类 → Supervisor 拆解 → 并行 Workers → 聚合 → LLM 回复
                                                      ↓
                                              高亮指令 → WS → 悬浮球/侧边栏
```

## 五、技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| LLM | Ollama + qwen2.5:7b | 本地运行，支持 Function Calling |
| Agent 框架 | LangGraph | 状态机编排，支持条件路由 |
| 向量库 | ChromaDB | 嵌入式，易部署 |
| 混合检索 | 向量 + BM25 + RRF | 兼顾语义和关键词 |
| 爬取 | Scrapling | 自适应 + 反检测 |
| 浏览器 | Edge (主) / Chrome / Firefox | Edge 是 Windows 默认 |
| 前端 | TypeScript + Vite | 类型安全，构建快 |
| 3D 渲染 | Three.js | 悬浮球动画 |
| 语音 | edge-tts + Vosk | 在线 TTS + 离线 ASR |

## 九、设计理念

1. **隐私优先** — 数据不出本机，采集即脱敏
2. **AI 原生** — 实时推理，不预录制
3. **自学习** — 元素指纹越用越准
4. **渐进披露** — 零配置可用，按需增强
5. **Edge 核心** — Windows 默认浏览器优先
6. **Karpathy 准则** — 简单优先，精准修改，目标驱动
