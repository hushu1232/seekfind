## 三、部署方式

### 3.1 方式一：单机部署（推荐）

每个员工电脑独立部署，适合 10 人以下团队。

**步骤：**

1. **下载安装包**
   - 从企业内部获取求问安装包
   - 解压到本地目录

2. **运行安装脚本**
   ```batch
   双击 install-enterprise.bat
   ```

3. **配置企业 API**（可选）
   - 安装脚本会询问是否配置企业 API
   - 输入企业 API 地址和 Key

4. **加载浏览器扩展**
   - Edge: edge://extensions → 加载已解压 → extension/dist
   - Chrome: chrome://extensions → 加载已解压 → extension/dist

5. **开始使用**
   - 打开任意网页
   - 点击求问悬浮球
   - 输入问题

### 3.2 方式二：批量部署

适合大规模部署（10+ 人），使用企业软件分发系统。

**步骤：**

1. **准备安装包**
   - 打包求问项目为 ZIP
   - 包含所有依赖和配置

2. **创建企业配置**
   - 编辑 `.env` 文件
   - 配置企业 API 和知识库

3. **分发安装包**
   - 通过企业软件分发系统（如 SCCM、Intune）
   - 或通过内部文件服务器

4. **静默安装**
   ```batch
   install-enterprise.bat /silent
   ```

### 3.3 方式三：集中部署

适合大型企业，所有服务集中在服务器。

**架构：**
- 后端服务部署在企业服务器
- 员工电脑只需安装浏览器扩展
- 通过 WebSocket 连接后端

**步骤：**

1. **部署后端服务**
   ```bash
   # 在服务器上
   docker compose up -d
   ```

2. **配置员工电脑**
   - 修改扩展配置，指向服务器地址
   - 或使用企业浏览器策略自动配置

## 四、企业 API 配置

### 4.1 支持的 API 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **OpenAI 兼容** | 标准 OpenAI API 格式 | https://api.openai.com/v1 |
| **通义千问** | 阿里云通义千问 API | https://dashscope.aliyuncs.com/api/v1 |
| **文心一言** | 百度文心一言 API | https://aip.baidubce.com/rpc/2.0/ai_custom/v1/wenxinworkshop |
| **企业自建** | 企业内部部署的 API | https://api.company.com/v1 |

### 4.2 配置示例

**OpenAI：**
```env
ENTERPRISE_API_BASE_URL=https://api.openai.com/v1
ENTERPRISE_API_KEY=sk-xxx
ENTERPRISE_MODEL=gpt-4o
```

**通义千问：**
```env
ENTERPRISE_API_BASE_URL=https://dashscope.aliyuncs.com/api/v1
ENTERPRISE_API_KEY=sk-xxx
ENTERPRISE_MODEL=qwen-max
```

**企业自建：**
```env
ENTERPRISE_API_BASE_URL=https://api.company.com/v1
ENTERPRISE_API_KEY=your-api-key
ENTERPRISE_MODEL=your-model-name
```

## 五、知识库管理

### 5.1 共享知识库

企业版支持多用户共享知识库，所有员工共享同一份文档。

**配置：**
```env
ENTERPRISE_SHARED_KNOWLEDGE=true
```

**优势：**
- 统一知识源
- 减少重复索引
- 便于维护更新

### 5.2 知识库更新

**手动更新：**
1. 管理员更新知识库文档
2. 重新构建索引
3. 员工电脑自动同步

**自动更新：**
```env
ENTERPRISE_AUTO_UPDATE=true
```

系统会定期检查知识库版本，自动更新。

### 5.3 企业内部文档导入

**支持格式：**
- URL（企业内部网页）
- 文本（操作手册）
- Markdown（技术文档）

**导入方式：**
```bash
# 通过 API 导入
curl -X POST http://localhost:8700/api/index/url \
  -H "Content-Type: application/json" \
  -d '{"url": "https://wiki.company.com/docs/xxx"}'

# 通过扩展导入
# 在侧边栏点击"导入"按钮
```

## 六、安全与隐私

### 6.1 数据安全

| 场景 | 处理方式 |
|------|---------|
| **本地模式** | 数据不出本机，完全私有 |
| **企业 API 模式** | 问题发送到企业 API，不发送到公网 |
| **隐私脱敏** | 密码、邮箱、手机号自动脱敏 |

### 6.2 企业 API 隐私警告

当降级到企业 API 时，系统会显示隐私警告：

```
⚠️ 本地模型不可用，已切换到企业 API
您的问题将发送到企业 API 服务器
请勿输入敏感信息（密码、银行卡号等）
```

### 6.3 审计日志（可选）

```env
ENTERPRISE_AUDIT_LOG=true
```

记录内容：
- 用户提问（脱敏后）
- 使用的模型（本地/企业）
- 检索的文档
- 高亮的元素

## 七、故障排查

### 7.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 无法连接后端 | Docker 未启动 | 运行 `docker compose up -d` |
| Ollama 未运行 | 服务未启动 | 运行 `ollama serve` |
| 模型未下载 | 首次使用 | 运行 `ollama pull qwen2.5:7b` |
| 企业 API 连接失败 | 配置错误 | 检查 API 地址和 Key |
| 检索结果不准确 | 知识库未索引 | 导入企业文档 |

### 7.2 诊断工具

**系统诊断：**
- 点击侧边栏右上角 🔍 按钮
- 查看各模块状态
- 一键修复常见问题

**API 诊断：**
```bash
# 检查后端健康
curl http://localhost:8700/health

# 检查企业 API
curl http://localhost:8700/api/enterprise/check

# 检查企业状态
curl http://localhost:8700/api/enterprise/status
```

### 7.3 日志查看

**后端日志：**
```bash
docker compose logs -f bff
```

**Ollama 日志：**
```bash
# Windows
%USERPROFILE%\.ollama\logs\server.log

# Mac/Linux
~/.ollama/logs/server.log
```
